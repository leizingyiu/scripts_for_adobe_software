# psd_link__batchConverToLinked

用于在 Photoshop 中批量扫描当前 PSD 内的**内嵌智能对象**，并将它们转换为**外链智能对象**。脚本会先做预检，再让你选择按“源 ID”或“内容”去重，最后把生成的外链文件输出到 PSD 同级目录下的 `_LinkedObjects` 文件夹。

## 1. 脚本解决什么问题

在复杂 PSD 里，经常会遇到这几种情况：

- 同一个内嵌智能对象被复制出很多层，修改同步困难。
- 内容完全相同，但并不是同一个共享源，导致文件重复。
- 文档里已经混有外链智能对象，人工逐个排查很费时间。

这个脚本的目标是：

- 自动扫描当前文档中的智能对象图层。
- 只处理“符合条件的内嵌智能对象”。
- 把它们批量转成外链，并统一输出为 `.psb` 文件。
- 支持两种去重方式：
  - **按源 ID 去重**：同一共享源只导出一次。
  - **按内容去重**：内容指纹相同的对象共用同一个外链文件。
- 可选给图层名和导出文件名增加 `[[id]]` 风格的标记，便于后续追踪与回链。

## 2. 运行前提

运行前请确认：

- Photoshop 已打开一个 PSD 文档。
- 当前活动文档已经保存到磁盘；未保存的文档会直接提示并退出。
- 文档中包含需要处理的智能对象图层。

脚本运行时会以**当前活动文档**为目标，并在该 PSD 所在目录创建：

- `_LinkedObjects`：保存最终生成的外链文件。
- `_LinkedObjects/_TempSmartObjectContents`：仅在你点击“保存临时文件”时生成。

## 3. 如何运行

1. 在 Photoshop 中打开目标 PSD。
2. 执行 `文件 -> 脚本 -> 浏览...`。
3. 选择 `psd_link__batchConverToLinked.jsx`。
4. 等待脚本完成扫描并弹出预检窗口。
5. 选择处理方式后执行。

如果你经常使用，也可以把脚本复制到 Photoshop 的 `Presets/Scripts` 目录，重启 Photoshop 后从脚本菜单直接运行。

## 4. 脚本实际处理规则

### 4.1 会处理哪些图层

脚本只会处理：

- 图层类型是 `Smart Object`。
- 被判定为**内嵌**的智能对象。
- 图层名和父组名没有命中跳过规则。

### 4.2 不会处理哪些图层

以下情况会被跳过：

- 已经有明确外链路径的智能对象。
- 名称命中 `SKIP_LAYER_OR_GROUP_NAME_CONTAINS` 的图层或组。
- 名称命中 `SKIP_LAYER_OR_GROUP_NAME_REGEX` 的图层或组。
- 当相关开关开启时，不在允许扩展名名单中的内嵌对象。

默认跳过关键字为：

```text
toLink_skip,link.js_skip
```

这表示只要图层名或组名包含这些片段，该图层或整个组都会被排除。

## 5. 预检窗口说明

脚本扫描后会弹出一个预检窗口，展示：

- 待处理图层总数。
- 按源 ID 去重后的独立数据源数量。
- 按内容去重后的独立数据源数量。
- 被跳过的图层/组及原因。
- 即将导出的目标目录。
- 图层清单，以及它们和“数据源代表图层”的关系。

窗口里有几个关键操作：

- **增加 Id 标记**：默认勾选。执行后会给图层名和导出文件名追加 ID 标签。
- **按源ID去重继续**：按智能对象源 ID 归并。
- **按内容去重继续**：按内容指纹归并。
- **保存临时文件**：把当前可处理智能对象的内部内容另存到临时目录，便于人工检查。
- **取消**：退出，不修改文档。

## 6. 两种去重模式

### 6.1 按源 ID 去重

适合以下情况：

- 图层本来就是复制出来的共享源。
- 你希望保留“同源关系”，不要把只是看起来相同、但来源不同的对象强行合并。

行为特点：

- 同一个 `smartObjectMore.ID` 只导出一次。
- 多个同源图层最终会链接到同一个生成的 `.psb` 文件。

### 6.2 按内容去重

适合以下情况：

- 多个智能对象来源不同，但内容完全相同。
- 你希望尽量减少重复的外链文件数量。

脚本的内容判定不是简单看名字，而是综合计算内容指纹，包括：

- 宽高
- 文档模式
- 位深
- 直方图 CRC
- 图层结构指纹 CRC

如果某些对象无法生成内容指纹，脚本会在预检中提示，并自动退回到“按源 ID”方式处理这些对象。

## 7. 输出结果

执行成功后，脚本会：

- 在 PSD 同级目录创建 `_LinkedObjects`。
- 为每个独立数据源导出一个 `.psb` 文件。
- 将所有匹配图层重新链接到对应外链文件。
- 可选地把图层名批量改成带 ID 标签的形式。
- 最后弹出统计结果和失败明细。

导出文件名基于“代表图层名”生成；如果重名，会自动追加 `_1`、`_2` 之类的后缀避免覆盖。

## 8. 图层命名与代表图层选择

脚本会尝试从同一数据源的多个图层中，选一个更像“原始命名”的图层作为代表名，用于：

- 生成外链文件名
- 在预检列表中标记“数据源”
- 在按内容去重时作为内容组的代表名称

默认会把下面这些词视为“复制痕迹”：

```text
拷贝,副本,Copy,copy
```

命中越少、名称越短、排序越靠前的图层，越可能被选为代表图层。

## 9. ID 标记规则

默认模板：

```text
[[{{id}}]]
```

默认 ID 来源：

```text
timestamp
```

可选值：

- `timestamp`：使用时间戳加计数器，适合一次性批量生成唯一标记。
- `layer.id`：尽量使用代表图层的数值 ID。

添加标记后，效果类似：

```text
按钮主视觉[[1779879747466_2]]
```

## 10. 头部配置项说明

脚本开头可直接修改以下常量：

| 配置项 | 作用 |
| --- | --- |
| `DEBUG` | 是否输出调试日志。 |
| `ENABLE_CONTENT_DEDUPE` | 是否启用按内容去重。 |
| `LAYER_NAME_ROOT_HINTS` | 判断“代表图层名”时用于识别副本命名的关键词。 |
| `ID_TAG_TEMPLATE` | 图层名和文件名的 ID 标签模板。 |
| `ID_TAG_ID_SOURCE` | ID 来源，支持 `timestamp` 或 `layer.id`。 |
| `SKIP_NON_PS_EMBEDDED_SMARTOBJECTS` | 是否跳过不在允许扩展名名单中的内嵌对象。 |
| `SKIP_NON_PS_EMBEDDED_SMARTOBJECTS_WITHOUT_OPEN` | 是否在不打开内容文档的情况下先做快速跳过判断。 |
| `ALLOWED_EMBEDDED_SMARTOBJECT_EXTENSIONS` | 允许作为 Photoshop 智能对象处理的扩展名。 |
| `ALLOWED_EMBEDDED_IMAGE_EXTENSIONS` | 允许作为图像型智能对象处理的扩展名。 |
| `SKIP_LAYER_OR_GROUP_NAME_CONTAINS` | 命中即跳过的名称片段，逗号分隔。 |
| `SKIP_LAYER_OR_GROUP_NAME_REGEX` | 命中即跳过的正则表达式，可写成 `/.../g` 形式。 |

默认允许的扩展名：

```text
psd,psb,psdt,png,jpg,jpeg,tif,tiff,gif,bmp,webp
```

## 11. 常见使用场景

### 场景 A：把共享内嵌智能对象全部转外链

推荐选择：

- 保持 `ENABLE_CONTENT_DEDUPE = true`
- 在预检窗口点击 **按源ID去重继续**

结果：

- 原本共享同一源的图层会统一链接到同一个 `.psb`。
- 只是名称不同但来源相同的图层，不会重复导出文件。

### 场景 B：把“内容完全一样但并非共享源”的对象也合并

推荐选择：

- 在预检窗口点击 **按内容去重继续**

结果：

- 只要内容指纹相同，就会复用同一个导出文件。
- 可减少多个重复 `.psb` 文件。

### 场景 C：先人工检查再决定

推荐选择：

- 先点击 **保存临时文件**
- 到 `_LinkedObjects/_TempSmartObjectContents` 检查导出的临时内容
- 再重新运行并决定使用哪种去重模式

## 12. 注意事项

- 脚本只基于**当前活动文档**运行，不会批量处理多个已打开文档。
- 文档必须先保存，否则无法确定外链输出目录。
- 脚本通过“是否存在外链路径”来判断智能对象是否为外链，这是当前版本修复误判的核心逻辑。
- 如果某些图层转换失败，脚本不会中断全部流程，而会在最后汇总失败详情。
- 文件名会经过非法字符清洗，但仍建议图层命名尽量简洁、稳定。

## 13. 常见问题

### Q1：运行后提示“未检测到任何符合条件的内嵌智能对象”

可能原因：

- 文档里没有智能对象。
- 文档中的智能对象本来就是外链。
- 图层或组名命中了跳过规则。
- 对象扩展名不在允许名单中。

### Q2：为什么有些图层没有被处理

请优先检查：

- 图层本身是否已经是外链。
- 图层名或父组名是否含有跳过关键字。
- 对应智能对象是否属于被排除的扩展名类型。

### Q3：为什么生成的是 `.psb` 而不是 `.psd`

当前主流程固定按 `.psb` 外链输出。  
“保存临时文件”功能在保存失败时会尝试回退为 `.psd`，但最终正式外链文件仍以 `.psb` 为主。

## 14. 建议工作流

1. 先复制一份 PSD 做测试。
2. 调整脚本头部的跳过规则和 ID 规则。
3. 运行脚本查看预检结果。
4. 如不确定，先导出临时文件检查。
5. 再选择“按源 ID”或“按内容”去重执行。
6. 完成后抽查几个关键图层，确认链接关系和命名是否符合预期。
