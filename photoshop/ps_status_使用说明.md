# PS\_STATUS（ps\_status.jsx）使用说明

这份脚本用于在 Photoshop 里用“状态层”批量生成/更新不同 UI 状态（例如：按钮默认/按下/禁用、多选组件不同组合等）。它不需要写代码，只需要按约定搭好图层结构，再用 Excel（TSV）批量喂数据即可。

***

## 第一部分：最简单用法（只学 Excel 一键创建并更新）

### 你需要准备什么

1. **一个 PSD**（打开着）。
2. **一个顶层 status 组**
   - 顶层（与其它大组同级）有且只有一个名为 `status` 的图层组。
   - 如果没有，脚本会自动创建；如果有多个同名 `status` 顶层组，会报错。
3. **若干个 Key（可选）**：用于让脚本知道“哪些地方需要切换”
   - 你可以先不做复杂结构，先从最常见的两种开始：
     - **开关/选项 Key：** 一个组，名字以英文冒号结尾，例如 `K_MULTI:`
     - **文本 Key：** 一个文本层，名字以英文冒号结尾，例如 `TITLE:`

### 一键操作流程（Excel -> PSD）

1. 在 Photoshop 中 **取消选中任何图层**（很重要）。
2. 运行脚本 `ps_status.jsx`。
3. 弹窗里选择 **“Excel 新建”**。
4. 在弹出的 **“PS\_STATUS Excel”** 窗口里：
   - `group 名称`：建议保持默认 `excel_status`（它会在 `status` 组里创建一个子组来装本次生成的状态层）。
   - 把 Excel 表格内容复制后粘贴到大文本框里（复制时是“按单元格”复制，脚本会按 Tab/换行解析）。
5. 点击 **“创建并更新”**。
6. 脚本会：
   - 在 `status/excel_status`（或你填写的 group 名称）里创建多条状态层；
   - 再根据每条状态层的配置，批量刷新生成最终画面。

### Excel 表格格式（最关键）

脚本读取的是 **TSV**（Tab 分隔）：

- **第一行是表头**
  - **第 1 列：状态名称**（只是名字，方便你认）
  - **第 2 列开始：Key 名称**（必须与 PSD 中的 key 名一致，大小写也要一致）
- **从第二行开始每行一个状态**
  - 第 1 列：状态名称
  - 第 2 列开始：对应 key 的 value

示例（可直接复制到 Excel 再复制出来）：

```tsv
状态名称	K_MULTI	K_TOGGLE	TITLE:
S_A	A	true	Hello
S_B	B	false	World
S_AB	A,B	-	Hello\\nWorld
```

### value 填写规则（不想踩坑就看这段）

- 空值：用 `-`（或留空）代表“这个 key 不生效/清空”
- 多选：用英文逗号 `,` 分隔，例如 `A,B`
- 开关值：推荐用 `true` / `false`（大小写无所谓）
- 选项值：必须能在 PSD 里找到同名选项图层（大小写敏感）
- 文本换行：在 Excel 里写 `\n`（两个字符），脚本会把它变成 Photoshop 的换行

## 第二部分：完整介绍（理解概念 + 能排查问题）

### 1. 脚本能做什么（3 种入口）

运行脚本后，根据你是否选中图层，会走不同模式：

1. **选中** **`status`** **组里的某些状态层/状态组**
   - 脚本只更新你选中的那一部分状态。
2. **选中一个包含多行/Tab 的文本层（TSV）**
   - 把文本层当作 TSV 输入，批量创建状态层。
3. **不选中任何图层**
   - 弹出菜单选择：
     - 更新全部状态层
     - 新建状态层（从当前画面反推配置）
     - Excel 新建（最推荐）

### 2. 核心概念：status 组 / 状态层 / key

#### 2.1 status 组

- 顶层必须有且只有一个名为 `status` 的组。
- 状态层通常都放在这个组里（脚本也会在这里新建）。

#### 2.2 状态层是什么

状态层本质是一个普通图层，但它的名字里包含一段配置：

- 只要图层名里出现 `{ ... }`，脚本就会把花括号中的内容当作配置解析
- 花括号外面的文字只是“人类可读的名字”

例如：

```text
B_MULTI_S_A={K_MULTI:A,K_TOGGLE:true,TITLE:Hello}
```

脚本更新状态层时，会根据这段配置去控制 key 目标图层的显示/文本，然后把最终画面“拍平”回写到状态层中。

#### 2.3 什么是 key

key 是“可切换的属性”。脚本会在 PSD 里扫描两类 key 目标：

1. **key 组**：名字以英文冒号结尾的图层组，例如 `K_MULTI:`
2. **key 文本层**：名字以英文冒号结尾的文本层，例如 `TITLE:`

Excel 的表头从第 2 列开始就是 key 名（例如 `K_MULTI`、`K_TOGGLE`、`TITLE:`），必须与 PSD 对得上。

### 3. key 的 3 种常见结构（决定 value 怎么写）

> 同一个 key 名在 PSD 里可以出现多处，但强烈建议它们的结构保持一致（否则会出现“混合类型”提示）。

#### 3.1 文本 Key（key 文本层）

- 结构：一个文本层，名字以冒号结尾，例如 `TITLE:`
- value 行为：
  - 空值（`-` / 空）：隐藏该文本层
  - 非空：显示该文本层，并把 value 写进文本内容
- 换行：Excel 里写 `\n`

#### 3.2 选项 Key（select，最常见）

- 结构：一个组 `K_MULTI:`，组里放选项子层（普通层/组都行），例如：`A`、`B`、`true`、`false`
- value 行为：
  - 空值：把所有选项子层都隐藏
  - `A`：只显示名为 `A` 的子层
  - `A,B`：同时显示 `A` 和 `B`（多选）
- 注意：选项匹配 **大小写敏感**，Excel 里写 `TRUE` 找不到 `true`。

#### 3.3 开关 Key（toggle）

开关是“同名子层触发”的特殊结构：

- 结构：一个组 `K_TOGGLE:`，组里存在一个（或多个）**名字恰好等于** **`K_TOGGLE`** 的子层
  - 这个同名子层就是“开关节点”
- value 行为：
  - `true`：显示同名子层
  - `false` 或空值：隐藏同名子层
  - 其它值：按默认当作 `true`（兼容脏数据）

> 这也是为什么有时你以为自己做的是 select，但脚本却把它识别成 toggle：只要组里不小心出现了一个同名子层，就会被判为 toggle。

#### 3.4 textContent Key（高级，用于“组内文本内容”）

这种结构用于“在一个 key 组里写文本”：

- 结构：组名 `NOTE:`，组里存在一个文本层名字恰好等于 `NOTE`
- value 行为：
  - 空值：整组隐藏
  - 非空：显示同名文本层并写入 value，同时其它非 key 子层一般也会显示（当作背景/装饰）

### 4. 常见提示与如何处理

#### 4.1 “找不到 value：XXX”

含义：你把某个 key 当作 select 用了，但在对应的 key 组选项里没有叫 `XXX` 的子层。

排查：

- 去 `file://...` 指向的 PSD 路径里看那一组
- 确认子层名是否大小写一致

#### 4.2 “value 不是 true/false/空值”

含义：这个 key 被识别为 toggle，但你填了 `A` / `B` 这类值。

排查：

- 去对应 key 组里找是否存在同名子层（例如组叫 `K_MULTI:`，组里有子层叫 `K_MULTI`）
- 如果你想要 select：删掉/改名那个同名子层

#### 4.3 “key(...) 在不同目标位置存在混合类型：...”

含义：同一个 key 名在 PSD 不同位置被做成了不同结构（比如一处是 select，一处是 toggle/textContent）。

建议：

- 同名 key 尽量保证结构一致
- 或者把其中一处改成不同 key 名（例如 `K_MULTI_A:` / `K_MULTI_B:`）

### 5. 小技巧（让批量更稳定）

- Excel 表头里 **Key 名不要随意改大小写**
- select 的 value 建议直接复制 PSD 里选项图层名（避免大小写错误）
- 需要换行的文本统一用 `\n`，不要在 Excel 单元格里真正换行（更稳）
- 如果弹窗提供 **“选中问题图层并退出”**：用它快速定位出错的那组图层
